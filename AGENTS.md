# AGENTS.md

このファイルは、Codex などのエージェントがこのリポジトリで作業する際のルールをまとめたものです。

## 目的
- 個人開発の VocabDock を安定して進める
- 変更方針とドキュメント更新のルールを明確にする

## プロジェクト前提
- 対象プラットフォーム: iOS / macOS
- 技術スタック: Swift + SwiftUI + SwiftData
- リポジトリ: 単一リポジトリ運用（`app/` 配下を本体として扱う）

## 作業ルール
- 回答は日本語で行う
- 既存の `note.md` / `prompt.md` は編集しない
- 個人メモは `__note/` 配下に置く
- `docs/` の更新は、会話で方針が確定した場合のみ行う
- 不要ファイルはコミット前に整理する

## 実装方針
- まず MVP を優先し、過剰な抽象化は避ける
- ただし将来 Android 展開を見据え、以下は仕様として固定する
  - 語彙データの論理モデル
  - API 正規化ルール
  - JSON エクスポート/インポート形式（`schema_version` を持つ）

## Git運用
- 作業ブランチは `codex/*` を使用する
- 変更は小さく区切ってコミットする
- コミット前に `git status` で不要差分を確認する

## 進捗管理
- タスク進捗は `docs/task_progress.md` で管理する
- 完了タスクはチェックボックスを `[x]` に更新する
