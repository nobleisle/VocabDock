# VocabDock 個人開発向け 開発・デプロイ手順（シンプル版）

## 1. MVPを固定する（30分）
- 初回リリースで必要な機能だけに絞る
- 対象例:
  - 単語/表現の検索
  - 検索結果のローカル保存
  - 再検索時のローカル表示
  - タグ/メモ
  - 履歴一覧

## 2. 技術を1回で決める（半日）
- アプリ実装は `Swift + SwiftUI` に固定
- ローカルDBは `SwiftData` に固定（必要に応じて Core Data へ移行可能）
- 外部取得APIはまず1つだけ採用（辞書APIを優先）
- 将来Android展開に備えて、以下を先に固定する
  - APIレスポンス正規化ルール
  - `VocabularyEntry` の論理モデル
  - JSONエクスポート/インポートのスキーマ

## 3. 縦スライスを先に作る（1-3日）
- 最短で以下を通す
  - 入力
  - API検索
  - 結果表示
  - ローカル保存
  - 同一語再検索時のローカル表示
- この流れが動けば土台完成

## 4. MVP機能を追加する（数日）
- タグ/メモの編集
- 履歴一覧と絞り込み
- オフライン時の保存済みデータ閲覧
- 余力があれば JSON エクスポート/インポート

## 5. リリース準備と配布
- 実機で最低1台ずつ確認（対象プラットフォーム）
- 既知不具合をメモ化
- テスト配布して使用感を確認
  - iOS: TestFlight
  - macOS: ローカル配布

## 技術選定の最短ルール
- 本プロジェクトは `Swift + SwiftUI + SwiftData + 辞書API1つ` で開始する
- 重要なのは最初の完成速度。拡張はMVPリリース後に行う

## 技術選定（確定）
- アプリ: Swift + SwiftUI（iOS / macOS）
- 状態管理: SwiftUI標準（`@State`, `@Observable`, `@Environment`）
- ローカルDB: SwiftData
- HTTP通信: URLSession
- セキュア保存: Keychain（APIキー格納用）
- 外部API: 辞書API 1系統（必要時のみLLMを追加）
- 将来拡張方針: UI実装は各OSで分離し、ドメイン仕様（モデル/JSON/API正規化）は共通化
- 配布:
  - iOS: TestFlight
  - macOS: ローカル配布（必要に応じてNotarization）
